Schedule

Weekly Sprints

Week 1

Environment Setup & Baseline Topologies

  • ติดตั้ง Linux (Ubuntu), ns-3 และโมดูล ndnSIM บนเครื่องของสมาชิกทุกคน
  • กำหนด topology เครือข่ายแบบ mesh (Omnitrix node เคลื่อนที่ 10 ตัว, Plumber consensus node คงที่ 3 ตัว)
  • ร่างการจำลอง TCP/IP พื้นฐาน เพื่อใช้เปรียบเทียบกับกรณีล้มเหลว
ผลลัพธ์: การจำลอง "Hello World" ที่ทำงานได้ใน ndnSIM
Week 2

Protocol Implementation (State Capsules)

  • สร้าง NDN Interest และ Data packet แบบกำหนดเอง ที่เป็นตัวแทน "State Capsules"
  • พัฒนาระบบ in-network caching เพื่อให้ node เคลื่อนที่สามารถแชร์ DNA packet กันได้โดยไม่ต้องผ่าน server กลาง
  • เขียนสคริปต์จำลองการที่ node ร้องขอการ transformation (ดึง DNA packet)
ผลลัพธ์: การแลกเปลี่ยน packet สำเร็จ ที่แสดงการ routing แบบอิงเนื้อหา
Week 3

Edge Cases, Security & Network Partitions

  • Scenario A: Network Partition — จำลองการตัดขาด link ที่ทำให้ Omnitrix node ถูกแยกออก; ตรวจสอบว่าการ transformation จากแคชยังใช้งานได้
  • Scenario B: Federated Consensus — จำลองการขอเอเลี่ยนตัวใหม่ พร้อมความหน่วงจากกระบวนการ PBFT consensus
ผลลัพธ์: ไฟล์ trace (.pcap / .tr) ที่บันทึก พฤติกรรมของเครือข่ายภายใต้สถานการณ์กดดัน
Week 4

Data Analysis, Reporting & Presentation

  • ดึงข้อมูลตัววัด (Throughput, Latency, Cache Hit Ratio) จากผลลัพธ์ ns-3
  • สร้างกราฟเปรียบเทียบ ONet mesh กับ TCP/IP มาตรฐาน
  • รวบรวมรายงานฉบับสมบูรณ์พร้อมแนวคิดเชิงทฤษฎีและผลการจำลอง
  • บันทึกวิดีโอนำเสนอ
ผลลัพธ์: เอกสารสถาปัตยกรรม, codebase ฉบับสมบูรณ์, กราฟตัววัด, และวิดีโอนำเสนอ
