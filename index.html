Project Overview

Why ONet?

Core Architecture

Bio-SCION Framework

ONet ยกเลิกการ routing แบบ IP มาตรฐาน และใช้แนวทางแบบไฮบริดที่รวม Information-Centric Networking กับ Isolated Trust Domains เข้าด้วยกัน

🛡️

SCION Isolation Domain (ISD)

ONet ทำงานเป็น SCION Isolation Domain แบบส่วนตัว ที่ถูกแยกออกจากอินเทอร์เน็ตสาธารณะด้วยการเข้ารหัส

  • ป้องกันการโจมตีแบบ DDoS และ BGP hijacking
  • แยกชั้นควบคุมทางชีวภาพที่สำคัญต่อชีวิตออกจากเครือข่ายสาธารณะ
  • ตรวจสอบเส้นทางด้วยการเข้ารหัส (Cryptographic Path Verification)
🔗

Named Data Networking (NDN)

การ routing อิงตามชื่อเนื้อหา (content names) แทนที่จะใช้ IP address ทางกายภาพ

  • ใช้ชื่อเนื้อหา เช่น /ONet/DNA/Tetramand/v1
  • ทนทานต่อความผิดพลาดสูง ด้วยระบบ peer caching
  • ไม่พึ่งพา server ตัวเดียว
📦

The "State Capsule" (Bio-Packet)

ใช้แทน TCP/IP packet แบบดั้งเดิม ด้วย NDN Data Packet แบบ atomic

  • Payload: ลำดับชีวภาพ (DNA) + นโยบายข้อจำกัด (ขีดจำกัดด้านสิ่งแวดล้อม)
  • Security: ทุก packet ถูกเซ็นด้วยการเข้ารหัสในระดับข้อมูล
  • Result: ข้อมูลตรวจสอบตัวเองได้ ไม่จำเป็นต้องเชื่อถือช่องทางการส่ง
Governance

Security Model

ระบบความปลอดภัยแบบ Capability-Based แบ่งเป็น 2 ระดับการทำงาน

Tier 1 — Local Authority

Emergency Actions

สำหรับการ Transformation แบบ zero-latency อุปกรณ์จะใช้ capability token ที่เก็บไว้ในแคชล่วงหน้า ไม่จำเป็นต้องเชื่อมต่อเครือข่าย — รับประกันความปลอดภัยในระหว่างที่เครือข่ายถูกแบ่งแยก

  • ใช้ capability token ที่แคชไว้ล่วงหน้า
  • ทำงานได้แม้ออฟไลน์ / ระหว่าง partition
  • ดำเนินการด้วย latency เป็นศูนย์
Tier 2 — Federated Consensus

Administrative Actions

การปลดล็อกลำดับ DNA ใหม่ ต้องอาศัยการลงมติแบบกระจายศูนย์ (PBFT) จาก Node ที่เชื่อถือได้ เพื่อตรวจสอบการปฏิบัติตามข้อกำหนดด้านจริยธรรม ก่อนออก capability token ใหม่

  • ใช้ระบบ Consensus แบบ PBFT กระจายศูนย์
  • ตรวจสอบการปฏิบัติตามข้อกำหนดด้านจริยธรรม
  • ระบบอนุรักษ์สายพันธุ์เอเลี่ยน (ASPS)
